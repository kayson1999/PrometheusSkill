# 查询脚本工具

## 🌐 Prometheus 查询地址说明

脚本的 `--url` 参数支持两种地址模式：

### 模式 1：直连 Prometheus 实例

直接连接 Prometheus 服务器，适用于可直接访问 Prometheus 的场景。

```bash
python3 scripts/prometheus_query.py --url http://prometheus:9090 --query 'up'
```

### 模式 2：通过 Grafana 数据源代理

部分环境的 Prometheus 数据通过 Grafana 数据源代理访问。Grafana 会将请求透明转发到后端 Prometheus 实例，支持标准的 Prometheus HTTP API。

**URL 拼接规则：**

```
http://<grafana_host>/api/datasources/proxy/uid/{数据源ID}
```

> 💡 **具体的集群地址和数据源 ID 由调用方（如集群信息 Skill）提供**，本技能不内嵌任何业务集群信息。

**使用示例：**

```bash
# 通过 Grafana 数据源代理查询
PROM_URL="http://grafana.example.com/api/datasources/proxy/uid/<数据源ID>"
python3 scripts/prometheus_query.py --url $PROM_URL \
  --mode range --range 1h \
  --query 'sum(rate(http_requests_total[5m]))' \
  --format chart --output /tmp/qps.html
```

> **注意**：Grafana 代理可能需要登录认证（Basic Auth）。如遇 401/403 错误，脚本会自动提示输入账号密码，也可通过 `--username`/`--password` 参数预先指定。

## Shell 脚本快速查询

```bash
#!/bin/bash
# prometheus_query.sh - Prometheus 快速查询工具
# 用法: ./prometheus_query.sh <prometheus_url> <query> [time_range] [step]

PROM_URL="${1:-http://localhost:9090}"
QUERY="$2"
TIME_RANGE="${3:-1h}"
STEP="${4:-60s}"

if [ -z "$QUERY" ]; then
  echo "用法: $0 <prometheus_url> <query> [time_range] [step]"
  echo "示例: $0 http://localhost:9090 'up' 1h 60s"
  exit 1
fi

# 计算时间范围
END=$(date +%s)
case "$TIME_RANGE" in
  *m) START=$((END - ${TIME_RANGE%m} * 60)) ;;
  *h) START=$((END - ${TIME_RANGE%h} * 3600)) ;;
  *d) START=$((END - ${TIME_RANGE%d} * 86400)) ;;
  *)  START=$((END - 3600)) ;;
esac

echo "=== Prometheus 查询 ==="
echo "URL: $PROM_URL"
echo "Query: $QUERY"
echo "Range: $TIME_RANGE (step: $STEP)"
echo ""

# 即时查询
echo "--- 即时查询结果 ---"
curl -s "${PROM_URL}/api/v1/query" \
  --data-urlencode "query=${QUERY}" | jq '.data.result[] | {metric: .metric, value: .value[1]}'

echo ""

# 范围查询
echo "--- 范围查询结果（最近 ${TIME_RANGE}）---"
curl -s "${PROM_URL}/api/v1/query_range" \
  --data-urlencode "query=${QUERY}" \
  --data-urlencode "start=${START}" \
  --data-urlencode "end=${END}" \
  --data-urlencode "step=${STEP}" | jq '.data.result[] | {metric: .metric, values_count: (.values | length), first: .values[0], last: .values[-1]}'
```

## Python 查询脚本

完整的 Python 查询脚本位于 `scripts/prometheus_query.py`，支持 8 种查询模式：

| 模式 | 功能 |
|------|------|
| `instant` | 即时查询（默认） |
| `range` | 范围查询（带自动 step 计算） |
| `targets` | 查看抓取目标状态 |
| `alerts` | 查看活跃告警（按严重级别分组） |
| `labels` | 查看标签名和值 |
| `rules` | 查看告警和 Recording Rules |
| `metadata` | 查看指标元数据 |
| `series` | 查找匹配的时间序列 |

## 使用脚本的示例

```bash
# 查看所有抓取目标状态
python3 scripts/prometheus_query.py --url http://localhost:9090 --mode targets

# 查看当前活跃告警
python3 scripts/prometheus_query.py --url http://localhost:9090 --mode alerts

# 查看所有 job 名称
python3 scripts/prometheus_query.py --url http://localhost:9090 --mode labels --label job

# 即时查询 CPU 使用率
python3 scripts/prometheus_query.py --url http://localhost:9090 \
  --query '100 - (avg by (instance) (rate(node_cpu_seconds_total{mode="idle"}[5m])) * 100)'

# 范围查询过去 6 小时的 QPS 趋势
python3 scripts/prometheus_query.py --url http://localhost:9090 \
  --mode range --range 6h --step 300s \
  --query 'sum(rate(http_requests_total[5m]))'

# 输出 JSON 格式
python3 scripts/prometheus_query.py --url http://localhost:9090 \
  --query 'up' --format json
```

## ⏰ 时间范围参数（--start / --end）

脚本支持 `--start` 和 `--end` 参数，可以查询**过去某天**或**指定时间范围**内的数据，而不仅限于"最近 N 小时"。

### 支持的时间格式

| 格式 | 示例 | 说明 |
|------|------|------|
| Unix 时间戳 | `1711500000` | 精确到秒的数字 |
| 日期 | `2024-03-27` 或 `2024/03/27` | 仅日期，自动设为 00:00:00 |
| 日期时间 | `2024-03-27 10:00:00` | 精确到秒 |
| ISO 格式 | `2024-03-27T10:00:00Z` | RFC3339 格式 |

### 优先级规则

| 参数组合 | 行为 |
|---------|------|
| `--start` + `--end` | 使用指定的绝对时间范围 |
| 仅 `--start`（纯日期） | 查询该天全天数据（00:00:00 ~ 23:59:59） |
| 仅 `--start`（含时分秒） | end = start + range（默认 1h） |
| 仅 `--end` | start = end - range（默认 1h） |
| 都不指定 | end = 当前时间, start = 当前时间 - range（原有行为） |

### 使用示例

```bash
# 查询昨天（2024-03-26）全天的 QPS 数据
python3 scripts/prometheus_query.py --url http://localhost:9090 \
  --mode range --start '2024-03-26' \
  --query 'sum(rate(http_requests_total[5m]))' --format chart

# 查询指定时间段的数据（上午 10 点到下午 6 点）
python3 scripts/prometheus_query.py --url http://localhost:9090 \
  --mode range --start '2024-03-26 10:00:00' --end '2024-03-26 18:00:00' \
  --query 'sum(rate(http_requests_total[5m]))' --format chart

# 查询某个时间点之前 2 小时的数据
python3 scripts/prometheus_query.py --url http://localhost:9090 \
  --mode range --range 2h --end '2024-03-26 18:00:00' \
  --query 'sum(rate(http_requests_total[5m]))'

# 查询从某个时间点开始的 6 小时数据
python3 scripts/prometheus_query.py --url http://localhost:9090 \
  --mode range --range 6h --start '2024-03-26 08:00:00' \
  --query 'sum(rate(http_requests_total[5m]))' --format chart

# 通过 Grafana 代理查询昨天的请求量
PROM_URL="http://grafana.example.com/api/datasources/proxy/uid/<数据源ID>"
python3 scripts/prometheus_query.py --url $PROM_URL \
  --mode range --start '2024-03-26' \
  --query 'sum by (job) (rate(http_requests_total[5m]))' \
  --format chart --output /tmp/qps-yesterday.html
```

> **提示**：`--start`/`--end` 参数仅在 `--mode range` 和 `--mode series` 下有效。即时查询（instant）使用 Prometheus 自身的 `time` 参数。

## 📊 HTML 图表输出模式

脚本支持 `--format chart` 参数，可将范围查询结果生成自包含的 HTML 交互式图表文件（基于 ECharts CDN，零额外依赖）。

### 图表特性

- 🎨 **深色主题**：类 Grafana 风格，视觉效果专业
- 🖱️ **交互式**：鼠标悬停显示数值、支持缩放和拖拽
- 📊 **统计卡片**：自动计算每条序列的最小值、最大值、平均值、最新值
- 📈 **多序列支持**：自动为每条序列分配颜色，图例可点击切换
- 🌐 **零依赖**：仅需浏览器打开，ECharts 通过 CDN 加载
- 📱 **响应式**：自适应窗口大小

### 使用示例

```bash
# 生成 QPS 趋势图表（默认输出到 /tmp/prom_chart_<timestamp>.html）
python3 scripts/prometheus_query.py --url http://localhost:9090 \
  --mode range --range 6h \
  --query 'sum(rate(http_requests_total[5m]))' \
  --format chart

# 指定输出路径
python3 scripts/prometheus_query.py --url http://localhost:9090 \
  --mode range --range 1h \
  --query 'sum by (job) (rate(http_requests_total[5m]))' \
  --format chart --output /tmp/qps-by-job.html

# 通过 Grafana 代理查询某服务请求量并生成图表
PROM_URL="http://grafana.example.com/api/datasources/proxy/uid/<数据源ID>"
python3 scripts/prometheus_query.py --url $PROM_URL \
  --mode range --range 1h \
  --query 'sum by (job) (rate(http_requests_total[5m]))' \
  --format chart --output /tmp/qps-by-job.html
```

### 参数说明

| 参数 | 说明 |
|------|------|
| `--format chart` | 启用 HTML 图表输出模式（仅 `range` 模式有效） |
| `--output <path>` | 指定 HTML 文件输出路径（可选，默认 `/tmp/prom_chart_<timestamp>.html`） |
| `--start <时间>` | 指定查询起始时间（可选，支持日期/日期时间/时间戳） |
| `--end <时间>` | 指定查询结束时间（可选，默认当前时间） |

> **注意**：`--format chart` 仅在 `--mode range` 下有效。即时查询（instant）不支持图表输出，因为只有单个时间点的数据。
> 
> **时间范围**：可通过 `--start`/`--end` 指定绝对时间范围，也可用 `--range` 指定相对时间范围。详见上方「⏰ 时间范围参数」章节。

## 🔐 认证支持

脚本支持 HTTP Basic Auth 认证，适用于需要登录的 Prometheus 实例。

### 认证参数

| 参数 | 说明 |
|------|------|
| `--username <user>` / `-U <user>` | Prometheus 登录账号（Basic Auth 用户名） |
| `--password <pass>` / `-P <pass>` | Prometheus 登录密码（Basic Auth 密码） |

### 认证行为说明

| 场景 | 行为 |
|------|------|
| 同时指定 `--username` 和 `--password` | 直接使用提供的凭据进行认证 |
| 仅指定 `--username`，未指定 `--password` | 交互式提示输入密码（密码不回显，更安全） |
| 未指定任何认证参数，但服务端返回 401/403 | 自动交互式提示输入账号和密码，然后重试请求 |
| 未指定认证参数，服务端无需认证 | 正常查询，无额外操作 |

### 认证使用示例

```bash
# 方式 1：命令行直接指定账号密码
 python3 scripts/prometheus_query.py --url http://prom:9090 \
  --username admin --password my_secret \
  --query 'up'

# 方式 2：指定账号，交互式输入密码（推荐，密码不会出现在命令历史中）
python3 scripts/prometheus_query.py --url http://prom:9090 \
  --username admin \
  --query 'up'
# 输出: 🔐 已指定账号 'admin'，请输入密码：
#    密码: ********

# 方式 3：不指定认证，服务端返回 401 时自动提示
python3 scripts/prometheus_query.py --url http://prom:9090 --query 'up'
# 输出: ⚠️  Prometheus 返回 401 Unauthorized，需要登录认证。
#    🔐 Prometheus 需要认证，请输入登录凭据：
#       账号: admin
#       密码: ********

# 带认证生成图表
python3 scripts/prometheus_query.py --url http://prom:9090 \
  --username admin --password my_secret \
  --mode range --range 1h \
  --query 'sum(rate(http_requests_total[5m]))' \
  --format chart --output /tmp/qps.html
```

> **安全提示**：推荐使用“方式 2”（仅指定 `--username`，交互式输入密码），避免密码出现在 shell 命令历史或进程列表中。
